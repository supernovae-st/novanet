## ADR-024: Trait Redefinition as "Data Origin"

### Changements Requis

| Avant | Après | Définition |
|-------|-------|------------|
| `invariant` | `defined` | Human creates ONCE (templates, configs) |
| `localized` | `authored` | Human writes PER locale (editorial) |
| `knowledge` (trait) | `imported` | External data brought in (corpora, APIs) |
| `generated` | `generated` | Our LLM produces (unchanged) |
| `aggregated` | `retrieved` | Fetched from external APIs (snapshots) |

### Distribution des Nodes par Trait

| Trait | Count | Exemples |
|-------|-------|----------|
| `defined` | 31 | Page, Block, Entity, PageStructure, BlockType, Locale, OrgConfig |
| `authored` | 2 | EntityContent, ProjectContent |
| `imported` | 22 | Term, Expression, Pattern, Culture, SEOKeyword, GEOQuery |
| `generated` | 5 | PageGenerated, BlockGenerated, OutputArtifact, PromptArtifact |
| `retrieved` | 3 | GEOAnswer, SEOKeywordMetrics, GEOMetrics |

## Notes

### Backward Compatibility Strategy

1. **YAML**: Serde aliases dans Rust pour anciens noms (`#[serde(alias = "invariant")]`)
2. **TypeScript**: KIND_META gardé (pas renommé en CLASS_TAXONOMY) jusqu'à v1.0.0
3. **Neo4j**: Migration script nécessaire pour bases existantes

### Risques

1. **Breaking change Neo4j**: Toutes les queries utilisant `:Meta:Kind` casseront
2. **Studio queries**: Vérifier que queryStore n'a pas de hardcoded labels
3. **Integration tests**: Nécessitent Neo4j running pour valider

### Timeline Suggérée

1. **Jour 1**: ADR-025 (création page-instruction, arcs, relations)
2. **Jour 2**: ADR-023 Neo4j (seeds)
3. **Jour 3**: ADR-023/024 Rust (TUI, generators, tests)
4. **Jour 4**: Documentation + verification finale
