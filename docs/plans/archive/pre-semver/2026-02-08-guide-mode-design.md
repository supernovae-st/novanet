# Guide Mode — Mission Control Design

**Date**: 2026-02-08
**Status**: Proposed
**Version**: v10.8

---

## Overview

The **Guide** mode is an interactive educational view in the NovaNet TUI that helps users understand the graph taxonomy through a NASA-style Mission Control dashboard.

---

## Current Navigation Modes

| Key | Mode | Purpose |
|-----|------|---------|
| `1` | **Meta** | Schema exploration (:Meta nodes — Realm, Layer, Trait, NodeKind, ArcKind) |
| `2` | **Data** | Instance exploration (actual data nodes — Locale, Page, Entity, etc.) |
| `3` | **Atlas** | Page composition view (how pages are assembled) |
| `4` | **Audit** | Data quality dashboard |
| `5` | **Guide** | *NEW* — Interactive taxonomy guide |

## The 5 Traits — Clarified Definitions

### Visual Encoding

| Trait | Symbol | Border Style | Role |
|-------|--------|--------------|------|
| **invariant** | ■ | solid | Structure that doesn't change between locales |
| **localized** | □ | dashed | OUTPUT — Generated content per locale (has invariant parent) |
| **knowledge** | ◊ | double | INPUT — Native locale knowledge (no parent, per-locale savoir) |
| **derived** | ═ | dotted | Computed/aggregated data |
| **job** | ○ | thin | Background tasks and processes |

### Key Distinction: Knowledge vs Localized

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  THE GENERATION PIPELINE                                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   KNOWLEDGE (INPUT)           INVARIANT              LOCALIZED (OUTPUT)     │
│   What we KNOW                What we STRUCTURE      What we GENERATE       │
│   ─────────────────           ─────────────────      ─────────────────      │
│   ◊ Term                      ■ Page                 □ PageL10n             │
│   ◊ Expression                ■ Entity               □ EntityL10n           │
│   ◊ Pattern                   ■ Block                □ BlockL10n            │
│   ◊ CultureRef                                                              │
│   ◊ Taboo                                                                   │
│   ◊ AudienceTrait                                                           │
│                                                                             │
│         │                           │                        ▲              │
│         │                           │                        │              │
│         ▼                           ▼                        │              │
│   ┌─────────────────────────────────────────────────────────────────┐       │
│   │                        LLM GENERATION                           │       │
│   │   Knowledge atoms + Invariant structure → Native content        │       │
│   └─────────────────────────────────────────────────────────────────┘       │
│                                                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│  BOTH ARE "PER-LOCALE" BUT WITH DIFFERENT ROLES:                            │
│                                                                             │
│  • Knowledge = INPUT (savoir)  — Loaded INTO the LLM as context             │
│  • Localized = OUTPUT (rendu)  — Generated BY the LLM as content            │
│                                                                             │
│  Knowledge nodes exist ONLY where needed (fr-FR may have 20K Terms,         │
│  sw-KE may have 500). They are native, not translated.                      │
│                                                                             │
│  Localized nodes exist for EVERY locale (200+ PageL10n per Page).           │
│  They are generated using knowledge + invariant structure.                  │
└─────────────────────────────────────────────────────────────────────────────┘
```

### The `*L10n` Suffix Rule

| Pattern | When to Use | Example |
|---------|-------------|---------|
| `FooL10n` | Node has invariant parent `Foo` | `PageL10n` → parent `Page` |
| `Foo` | Standalone (no parent) | `Term`, `Expression` (knowledge atoms) |

**Why no `TermL10n`?** Terms ARE locale-native. There's no invariant "Term" that gets localized. A Term in fr-FR is born in fr-FR.

**Why `EntityL10n`?** Entity is invariant (product/brand identity doesn't change). The localized content (descriptions, taglines) is generated per locale.

## Ludic & Interactive Elements

### Visual Feedback

1. **Smooth Transitions**
   - Panel slide animation when switching traits
   - Highlight pulse on selection
   - Breadcrumb trail showing navigation path

2. **Color-Coded Everything**
   - Selected trait has bright border, others dimmed
   - Progress bars for stats (not just numbers)
   - Consistent use of taxonomy colors

3. **Discovery Elements**
   - "Did you know?" tips that rotate
   - Relationship hints: "INVARIANT nodes always have LOCALIZED pairs"
   - Count animations when drilling down

### Quick Jump Shortcuts

| Shortcut | Action |
|----------|--------|
| `gi` | Go to INVARIANT trait |
| `gl` | Go to LOCALIZED trait |
| `gk` | Go to KNOWLEDGE trait |
| `gd` | Go to DERIVED trait |
| `gj` | Go to JOB trait |

### Responsiveness Principles

- Instant feedback on every keypress
- Never feel stuck (always show available actions)
- Always know where you are (breadcrumbs)
- Always know what you can do (shortcuts visible)
