### Block Node Classes Summary

| Node Class | Layer | Purpose |
|------------|-------|---------|
| **Block** | org/structure | Structural slot on a Page (position, anchor_id) |
| **BlockType** | org/instruction | Reusable JSON schema template (hero, faq, cta...) |
| **BlockInstruction** | org/instruction | Generation prompt with @entity: and @link: refs |
| **BlockRules** | org/instruction | Gherkin constraints, versioned, A/B testable |
| **BlockNative** | org/output | Generated content (payload JSON per locale) |

### Arc Relationships

| Arc | Source | Target | Description |
|-----|--------|--------|-------------|
| `[:HAS_BLOCK {order}]` | Page | Block | Page contains ordered Blocks |
| `[:OF_TYPE]` | Block | BlockType | Block uses this schema |
| `[:HAS_INSTRUCTION]` | Block | BlockInstruction | Block has generation prompt |
| `[:HAS_RULES]` | BlockType | BlockRules | Type defines constraints |
| `[:HAS_NATIVE]` | Block | BlockNative | Block has native content |
| `[:FOR_LOCALE]` | BlockNative | Locale | Native targets locale |

## Summary: 6 Key Rules

```
╔═══════════════════════════════════════════════════════════════════════════════╗
║  6 KEY RULES — NovaNet Ontology                                               ║
╠═══════════════════════════════════════════════════════════════════════════════╣
║                                                                               ║
║  1. ENTITY owns SEMANTICS, PAGE owns URL                                      ║
║     └── Entity.key = "entity:qr-code" (concept)                               ║
║     └── Page.key = "page:qr-code-generator" (structure)                       ║
║     └── PageNative.slug = from EntityNative.denomination_forms.url            ║
║                                                                               ║
║  2. denomination_forms are PRESCRIPTIVE, not descriptive                      ║
║     └── LLM MUST use these forms, cannot invent alternatives                  ║
║     └── text/title/abbrev/url — each has specific usage context               ║
║     └── "QR code" (French) NOT "code QR" — semantic accuracy matters          ║
║                                                                               ║
║  3. GENERATION, not TRANSLATION                                               ║
║     └── Content is natively generated per locale                              ║
║     └── LocaleVoice, LocaleCulture inform tone and style                      ║
║     └── Expressions, Patterns provide locale-specific vocabulary              ║
║                                                                               ║
║  4. *Native pattern for locale content                                        ║
║     └── Entity → EntityNative (authored)                                      ║
║     └── Page → PageNative (generated)                                         ║
║     └── Block → BlockNative (generated)                                       ║
║     └── All connected via [:HAS_NATIVE] arc                                   ║
║                                                                               ║
║  5. Block = Structure + Type + Instruction → Native                           ║
║     └── Block: position, references type and instruction                      ║
║     └── BlockType: JSON schema for output structure                           ║
║     └── BlockInstruction: prompt guidance for LLM                             ║
║     └── BlockNative: generated content following schema                       ║
║                                                                               ║
║  6. Page [:ABOUT] Entity with roles and weights                               ║
║     └── role: "focus" (weight=1.0) — primary entity, determines slug          ║
║     └── role: "related" (weight<1.0) — secondary context                      ║
║     └── Multiple entities enrich LLM context                                  ║
║                                                                               ║
╚═══════════════════════════════════════════════════════════════════════════════╝
```
