## Design Decisions

### 1. Hybrid Data Strategy

**Decision**: Extract structured properties + preserve raw_markdown for LLM context.

**Rationale**:
- Consumer is an LLM → narrative context is valuable ("NEVER use X", cultural notes)
- Structured props enable queries ("all locales with decimal_separator = ','")
- `raw_markdown` captures what can't be structured (decision algorithms, examples)

**Rule**:
- Factual + queryable → structured property
- Contextual + narrative → raw_markdown
- Closed lists → JSON property

### 2. Schema Improvements

#### Formatting (+7 properties)

| Property | Type | Description |
|----------|------|-------------|
| `month_names` | json | Full month names in locale language |
| `month_names_abbr` | json | Abbreviated month names |
| `day_names` | json | Full day names (Monday first) |
| `day_names_abbr` | json | Abbreviated day names |
| `street_types` | json | Common street type words |
| `postal_code_format` | string | Postal code format description |
| `national_id_patterns` | json | Country-specific ID formats (NIR, SIREN, SIRET, IBAN) |

#### Adaptation (+5 properties)

| Property | Type | Description |
|----------|------|-------------|
| `punctuation_spacing` | json | Space rules before/after punctuation |
| `typography_conventions` | json | Quotes, dashes, ellipsis conventions |
| `technical_terms_preferred` | json | Preferred local terms over English |
| `technical_terms_accepted` | json | English terms accepted in locale |
| `legal_compliance` | json | Key legal frameworks |

#### Slugification (+1 property)

| Property | Type | Description |
|----------|------|-------------|
| `stopwords_by_category` | json | Stopwords by grammatical category (optional, additive) |

**Note**: Slugification data is already well-structured. Do NOT modify existing data.

### 3. Parser Architecture

```
src/parsers/
├── locale_rules.rs          # Module principal + shared utilities
├── formatting_parser.rs     # Parse 2-rules-formatting/*.md
├── adaptation_parser.rs     # Parse 2-rules-adaptation/*.md
└── slugification_parser.rs  # Parse 2-rules-slug/*.md
```

Each parser:
1. Reads MD file
2. Extracts structured properties (markdown tables → JSON)
3. Preserves raw_markdown (complete or relevant sections)
4. Returns typed Rust struct

### 4. Output Strategy

**Granular Cypher files** (one per rule type):

```
packages/db/seed/
├── 20-locales.cypher           # Existing (Locale nodes + FALLBACK_TO)
├── 21-formatting.cypher        # NEW: Formatting nodes + HAS_FORMATTING
├── 22-slugification.cypher     # NEW: Slugification nodes + HAS_SLUGIFICATION
└── 23-adaptation.cypher        # NEW: Adaptation nodes + HAS_ADAPTATION
```

### 5. CLI Command

```bash
# Generate all rule types
cargo run -- locale generate-rules \
  --source-dir=/path/to/localization-data \
  --output-dir=packages/db/seed

# Generate specific type
cargo run -- locale generate-rules --type=formatting ...

# Dry run (preview without writing)
cargo run -- locale generate-rules --dry-run ...
```

## Cypher Output Format

### Example: 21-formatting.cypher

```cypher
// ═══════════════════════════════════════════════════════════════════════════════
// 21-formatting.cypher - Generated by `novanet locale generate-rules`
// v10.6 - 200 Formatting nodes
// ═══════════════════════════════════════════════════════════════════════════════

// ─── Formatting Nodes ────────────────────────────────────────────────────────

MERGE (f:Formatting {key: 'fr-FR'})
ON CREATE SET
  f.display_name = 'French (France) Formatting',
  f.description = 'Date, number, currency formatting for fr-FR',
  f.llm_context = 'USE: numeric/date/currency formatting...',
  f.decimal_separator = ',',
  f.thousands_separator = ' ',
  f.date_format = 'DD/MM/YYYY',
  f.time_system = '24h',
  f.currency_code = 'EUR',
  f.currency_symbol = '€',
  f.currency_position = 'after',
  f.month_names = '["janvier","février",...]',
  f.day_names = '["lundi","mardi",...]',
  f.street_types = '["rue","avenue","boulevard",...]',
  f.validation_patterns = '{...}',
  f.raw_markdown = '...',
  f.created_at = datetime(),
  f.updated_at = datetime()
ON MATCH SET
  f.updated_at = datetime();

// ─── HAS_FORMATTING Arcs ─────────────────────────────────────────────────────

MATCH (l:Locale {key: 'fr-FR'}), (f:Formatting {key: 'fr-FR'})
MERGE (l)-[:HAS_FORMATTING]->(f);
```
