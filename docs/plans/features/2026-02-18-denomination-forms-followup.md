# ADR-033: Denomination Forms

**Status**: Approved (v0.13.1)

**Problem**: LLMs generating content for Entity/EntityNative have no canonical
reference for how to name the entity in each context:
- When to use "qr code" vs "QR Code" vs "qr" vs "QR"
- How to derive the URL slug from keyword data
- How to write cross-locale internal link anchors consistently
- How to handle native-script locales (zh-CN: 二维码 vs QR码 vs QR Code)

**Decision**: Add `denomination_forms` — a prescriptive typed array on Entity and
EntityNative — as the single naming authority for LLMs.

## ABSOLUTE RULE

**The LLM MUST use ONLY values from denomination_forms when referring to an entity.**
No invention, paraphrase, or unlisted variation is allowed.

## Form Types

| type   | On Entity | On EntityNative | When to use |
|--------|-----------|-----------------|-------------|
| text   | ✅        | ✅              | Prose and body content |
| title  | ✅        | ✅              | H1, H2, meta_title |
| abbrev | ✅        | ✅              | After first mention, short text |
| mixed  | ❌        | native_script only | Tech/brand hybrid (e.g. "QR码" for zh-CN) |
| base   | ❌        | native_script only | International reference (e.g. "QR Code" in zh-CN content) |
| url    | ❌        | ✅ post-pipeline | URL-safe slug (see below) |

### Rules per type

- **text**: lowercase, locale-appropriate
- **title**: Title Case or locale Title convention
- **abbrev**: shortest recognized form
- **mixed**: only for `native_script` locales (ja-JP, ko-KR, zh-CN, ar-SA)
- **base**: only for `native_script` locales — the international reference form
- **url**: see URL Form section below

## URL Form (url type)

`url` on Entity: **DOES NOT EXIST** — English URL lives on `Page.slug` (ADR-030).

`url` on EntityNative: populated POST-SEO pipeline, NEVER set manually.

### Locale slugification coupling (ADR-032)

| Rule | Locales | Effect | Example |
|------|---------|--------|---------|
| latin_strip | en-US, en-GB | Remove diacritics | café → cafe |
| latin_preserve | fr-FR, es-MX, es-ES, pt-BR | Keep diacritics | código-qr ✓ |
| latin_transform | de-DE, de-AT | ü→ue, ö→oe, ä→ae, ß→ss | für → fuer |
| transliterate | ru-RU, uk-UA, el-GR | Cyrillic/Greek → Latin | код → kod |
| native_script | ja-JP, ko-KR, zh-CN, ar-SA | ALWAYS romanized ASCII | qr-code-sakusei |

### Feedback loop (ADR-030)

```
SEOKeyword.slug_form (INPUT REFERENCE)
    │
    └──[derivation: copy/extract/merge/modify/derive]──>
         BlockNative:head-seo-meta.slug (DERIVED, ADR-030)
              │
              └──[DERIVED_SLUG_FROM]──>
                   EntityNative.denomination_forms[url] (WRITTEN BACK)
```

After the SEO pipeline runs, the derived slug is written back to
`EntityNative.denomination_forms[url]`. This enables cross-page URL consistency:
any page linking to this entity can use `denomination_forms[url]` to build the href.

## Two Distinct Systems

| System | Where | Decided by |
|--------|-------|------------|
| `denomination_forms` | Entity / EntityNative | Editorial (human) |
| Slug URL | BlockNative:head-seo-meta | SEO pipeline (LLM + strategy) |

The SEO-derived slug is written back to EntityNative.denomination_forms[url].

## Priority Field (EntityNative only)

```
priority: 1  → LLM default (use this first)
priority: 2  → Acceptable variation within same paragraph
priority: 3+ → Rare or context-specific alternative
```

## Validation Rules

1. Entity MUST have: `text` + `title` + `abbrev` (3 minimum)
2. EntityNative MUST have: `text` + `title` + `abbrev` (3 minimum)
3. `url` is optional — populated by SEO pipeline, never set manually
4. `mixed` / `base` only allowed for `native_script` locales
5. `url` values MUST follow locale slugification rules (ADR-032)
6. native_script locales: `url` is ALWAYS romanized ASCII (never native characters)

## Relation to Other ADRs

| ADR | Relation |
|-----|----------|
| ADR-029 (*Native Pattern) | denomination_forms lives on EntityNative — follows *Native convention |
| ADR-030 (Slug Ownership) | `url` form is the feedback channel from the SEO pipeline |
| ADR-032 (URL Slugification) | Governs which locale rules apply to `url` form values |
| ADR-028 (Page-Entity) | denomination_forms used in @ references and internal link anchors |

/// Nodes that MUST have denomination_forms property (ADR-033).
const DENOMINATION_FORMS_NODES: &[&str] = &["Entity", "EntityNative"];

// Rule 14: denomination_forms must be present (ADR-033)
if DENOMINATION_FORMS_NODES.contains(&node.def.name.as_str()) {
    let has_denomination_forms = node.def.properties
        .as_ref()
        .map(|p| p.contains_key("denomination_forms"))
        .unwrap_or(false)

            .map(|p| p.contains_key("denomination_forms"))

    if !has_denomination_forms {
        issues.push(SchemaIssue {
            node_name: node.def.name.clone(),
            severity: IssueSeverity::Error,
            rule: "DENOMINATION_FORMS_REQUIRED",
            message: "Entity and EntityNative must define 'denomination_forms' property (ADR-033)".into(),
        });
    }

  - id: denomination-forms-context
    name: Denomination Forms Context
    description: Load denomination_forms for an entity in a specific locale — used by MCP agents for naming authority
    category: contextual
    icon: { web: type, terminal: "Ⅹ" }
    color: "#94a3b8"
    root_type: EntityNative
    contextual: true
    cypher: |
      // denomination-forms-context
      // Load canonical naming forms for entity in locale
      // Params: $entityKey (e.g. "qr-code"), $localeKey (e.g. "fr-FR")
      MATCH (e:Entity {key: $entityKey})
      OPTIONAL MATCH (e)-[:HAS_NATIVE]->(en:EntityNative {locale_key: $localeKey})
      OPTIONAL MATCH (en)-[:FOR_LOCALE]->(l:Locale)
      WITH e, en, l
      RETURN
        e.key AS entity_key,
        e.denomination_forms AS entity_forms,
        en.key AS entity_native_key,
        en.locale_key AS locale_key,
        en.denomination_forms AS locale_forms,
        l.slugification AS slugification_rule
      LIMIT 1
```
