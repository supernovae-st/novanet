## Workflow

### Full Reset (dev)

```bash
pnpm infra:reset   # Drops volume, recreates, runs all seeds
```

### Incremental Seed

```bash
pnpm infra:seed    # Runs all seeds (MERGE = idempotent)
```

### Backup Before Changes

```bash
pnpm db:backup -- --name before-my-changes
```

### Restore If Broken

```bash
pnpm db:restore backups/neo4j-before-my-changes.tar.gz
```

## Adding New Data

### New Locale

1. Add locale to CSV sources: `packages/core/models/data/locales/*.csv`
2. Add locale MD files: `packages/core/models/data/locales/*.md`
3. Regenerate: `novanet locale generate`
4. Run `pnpm infra:seed`

### New Organization

1. Copy `30-org-template.cypher` to `30-org-mycompany.cypher`
2. Update keys and properties
3. Run `pnpm infra:seed`

### New Project

1. Create `31-project-myproject.cypher`
2. Link to Organization via `[:HAS_PROJECT]`
3. Add Pages, Entities as needed
4. Run `pnpm infra:seed`

## Best Practices

1. **Use MERGE** - Idempotent inserts, safe to re-run
2. **Use keys** - Every node needs a unique `key` property
3. **Link to Kinds** - Use `[:OF_KIND]` or let `99-autowire-kinds.cypher` handle it
4. **Order matters** - Lower numbers run first, dependencies must exist
5. **Git track everything** - Seeds are code, review changes
6. **Backup before migrations** - Use `pnpm db:backup`

## Verification

After seeding, verify counts:

```cypher
// Count by Realm
MATCH (n)-[:OF_KIND]->(:Kind)-[:IN_REALM]->(r:Realm)
RETURN r.key as realm, count(n) as nodes
ORDER BY realm;

// Count by Layer
MATCH (n)-[:OF_KIND]->(:Kind)-[:IN_LAYER]->(l:Layer)
RETURN l.key as layer, count(n) as nodes
ORDER BY layer;
```
