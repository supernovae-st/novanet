# v7.11.0 Design: Simplification + Versioning

**Date**: 2026-01-29
**Status**: Approved
**Author**: Thibaut + Claude

## Summary

Simplify standard properties (remove unused fields) and add generation versioning for BlockL10n/PageL10n.

## Changes Overview

### 1. Standard Properties Simplification

**Remove from all nodes:**
- `priority` - YAGNI, context budgeting never implemented
- `freshness` - YAGNI, cache managed at app level, we have `updated_at`
- `icon` - UI concern, managed in Studio not graph

**New standard properties (6 fields, was 9):**
```yaml
standard_properties:
  key: string              # Unique identifier
  display_name: string     # Human-readable name
  description: string      # Short description
  llm_context: string      # LLM generation context
  created_at: datetime
  updated_at: datetime
```

### 2. Generation Versioning

Add version tracking to BlockL10n and PageL10n for:
- **Comparison**: A/B testing between versions
- **Audit**: Track who/when/why regenerated
- **Debugging**: Understand generation issues
- **Structure history**: Which blocks composed a page at time T

#### BlockL10n Changes

**New properties:**
```yaml
version: int              # 1, 2, 3...
published_at: datetime    # When put live
replaced_at: datetime?    # When replaced (null = current)
```

**New relation:**
```yaml
PREVIOUS_VERSION:
  from: BlockL10n
  to: BlockL10n
  cardinality: "1:1"
  description: "Links to previous version in history chain"
```

#### PageL10n Changes

**New properties:**
```yaml
version: int              # 1, 2, 3...
published_at: datetime    # When put live
replaced_at: datetime?    # When replaced (null = current)
```

**New relation:**
```yaml
PREVIOUS_VERSION:
  from: PageL10n
  to: PageL10n
  cardinality: "1:1"
  description: "Links to previous version in history chain"
```

### 3. Remove PageMetrics

**Reason**: YAGNI - metrics can be queried from GA/PostHog using `published_at`/`replaced_at` date ranges. No need to duplicate in Neo4j.

**Can be re-added later** if we need caching/aggregation.

## Graph Model

```
Page ──[:HAS_OUTPUT]──▶ PageL10n (v2, current)
  │                         │
  │                         ├──[:ASSEMBLES]──▶ BlockL10n hero-v2
  │                         ├──[:ASSEMBLES]──▶ BlockL10n features-v1
  │                         ├──[:FOR_LOCALE]──▶ Locale fr-FR
  │                         │
  │                         └──[:PREVIOUS_VERSION]
  │                                    │
  │                                    ▼
  │                            PageL10n (v1)
  │                                 │
  │                                 ├──[:ASSEMBLES]──▶ BlockL10n hero-v1
  │                                 └──[:ASSEMBLES]──▶ BlockL10n features-v1
  │
  └──[:HAS_BLOCK]──▶ Block ──[:HAS_OUTPUT]──▶ BlockL10n (v2, current)
                                                   │
                                                   └──[:PREVIOUS_VERSION]
                                                              │
                                                              ▼
                                                      BlockL10n (v1)
```

## Key Rules

1. **HAS_OUTPUT always points to current** (`replaced_at = null`)
2. **PREVIOUS_VERSION is auxiliary** - excluded from spreading activation
3. **Generation queries ignore PREVIOUS_VERSION** - only use HAS_OUTPUT
4. **ASSEMBLES captures structure** - each PageL10n version links to its BlockL10n versions

## Example Queries

```cypher
// Current page with blocks
MATCH (p:Page {key: $key})-[:HAS_OUTPUT]->(pl:PageL10n)
WHERE pl.replaced_at IS NULL
MATCH (pl)-[:ASSEMBLES]->(bl:BlockL10n)
RETURN pl, collect(bl)

// Page history
MATCH (p:Page {key: $key})-[:HAS_OUTPUT]->(current:PageL10n)
MATCH path = (current)-[:PREVIOUS_VERSION*0..]->(old:PageL10n)
RETURN old.version, old.published_at, old.replaced_at
ORDER BY old.version DESC

// Compare versions (which blocks changed)
MATCH (pl1:PageL10n {version: 1})-[:ASSEMBLES]->(bl1:BlockL10n)
MATCH (pl2:PageL10n {version: 2})-[:ASSEMBLES]->(bl2:BlockL10n)
RETURN pl1.version, collect(bl1.key), pl2.version, collect(bl2.key)

// Get metrics from GA for a version
// Use published_at and replaced_at as date range filter in GA API
MATCH (pl:PageL10n {version: $version})
RETURN pl.published_at, pl.replaced_at
// Then query GA with these dates
```
