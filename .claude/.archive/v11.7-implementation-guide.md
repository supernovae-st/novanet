# v11.7 Implementation Guide

This guide provides step-by-step instructions for implementing the v11.7 Unified Tree architecture.

## Quick Start

```bash
# 1. Verify environment
pnpm infra:up && pnpm infra:seed
cargo nextest run && pnpm test

# 2. Create isolated worktree
git worktree add ../novanet-v11.7 -b feature/v11.7-unified-tree

# 3. Start Phase 0
cd ../novanet-v11.7
# Run Neo4j migration...
```

## Architecture Overview

v11.7 introduces a **Unified Tree** that combines meta-graph and instance hierarchy:

```
Realm (shared/org)
  └── Layer (config/semantic/output/...)
       └── Class (Page/Entity/Block/...)
            └── Instance (page:homepage, entity:qr-code/...)
```

Key changes:
- **Neo4j relationships**: HAS_LAYER, HAS_CLASS, BELONGS_TO_FAMILY
- **Dual icons**: web (Lucide) + terminal (Unicode) at all levels
- **Lazy loading**: Instances fetched on-demand
- **Unified types**: Same data structure for TUI and Studio

### Phase 0: Neo4j Migration

**Goal**: Add structural relationships between Realm, Layer, Class, ArcFamily, and ArcClass nodes.

- [ ] Create `packages/db/seed/migrations/v11.7-unified-tree.cypher`
- [ ] Add HAS_LAYER relationships (Realm → Layer)
- [ ] Add HAS_CLASS relationships (Layer → Class)
- [ ] Add BELONGS_TO_FAMILY relationships (ArcClass → ArcFamily)
- [ ] Add performance index for instances
- [ ] Validate with verification query
- [ ] Test: `cargo run -- meta` shows correct counts

**Migration file template**:
```cypher
// v11.7 Unified Tree Migration
// Adds structural relationships for tree hierarchy

// 1. Create HAS_LAYER relationships
MATCH (r:Realm), (l:Layer)
WHERE l.realm = r.key
MERGE (r)-[:HAS_LAYER]->(l);

// 2. Create HAS_CLASS relationships
MATCH (l:Layer), (c:Class)
WHERE c.layer = l.key AND c.realm = l.realm
MERGE (l)-[:HAS_CLASS]->(c);

// 3. Create BELONGS_TO_FAMILY relationships
MATCH (a:ArcClass), (f:ArcFamily)
WHERE a.family = f.key
MERGE (a)-[:BELONGS_TO_FAMILY]->(f);

// 4. Performance index for instance loading
CREATE INDEX instance_kind_idx IF NOT EXISTS FOR (n:Node) ON (n.kind);
```

**Verification query**:
```cypher
MATCH (r:Realm)-[:HAS_LAYER]->(l:Layer)-[:HAS_CLASS]->(c:Class)
RETURN r.key AS realm, count(DISTINCT l) AS layers, count(DISTINCT c) AS classes
```

fn render_tree_line(node: &UnifiedNode, indent: usize, is_selected: bool) -> Line {
    let icon = node.icon_terminal;
    let badges = node.badges.iter()
        .map(|b| Span::styled(format!(" [{}]", b.text), Style::default().fg(parse_color(&b.color))))
        .collect::<Vec<_>>();

    Line::from(vec![
        Span::raw(" ".repeat(indent * 2)),
        Span::raw(format!("{} ", icon)),
        Span::styled(&node.label, if is_selected { Style::default().reversed() } else { Style::default() }),
    ]).into_iter().chain(badges).collect()
}

**Tree store** (`apps/studio/src/stores/treeStore.ts`):
```typescript
import { create } from 'zustand';
import type { UnifiedTreeNode } from '@novanet/core';

interface TreeState {
  root: UnifiedTreeNode[];
  expandedIds: Set<string>;
  selectedId: string | null;
  loadingIds: Set<string>;

  toggleExpand: (id: string) => void;
  selectNode: (id: string) => void;
  loadChildren: (id: string) => Promise<void>;
}

export const useTreeStore = create<TreeState>((set, get) => ({
  root: [],
  expandedIds: new Set(),
  selectedId: null,
  loadingIds: new Set(),

  toggleExpand: (id) => set((state) => {
    const expanded = new Set(state.expandedIds);
    if (expanded.has(id)) {
      expanded.delete(id);
    } else {
      expanded.add(id);
    }
    return { expandedIds: expanded };
  }),

  selectNode: (id) => set({ selectedId: id }),

  loadChildren: async (id) => {
    set((state) => ({ loadingIds: new Set([...state.loadingIds, id]) }));
    try {
      const response = await fetch(`/api/tree/${id}/children`);
      const children = await response.json();
      // Update tree with children
    } finally {
      set((state) => {
        const loading = new Set(state.loadingIds);
        loading.delete(id);
        return { loadingIds: loading };
      });
    }
  },
}));
```

**Tree node component** (`apps/studio/src/components/tree/UnifiedTreeNode.tsx`):
```tsx
import { ChevronRight, ChevronDown, Loader2 } from 'lucide-react';
import * as Icons from 'lucide-react';
import type { UnifiedTreeNode as TreeNode } from '@novanet/core';
import { useTreeStore } from '@/stores/treeStore';

interface Props {
  node: TreeNode;
  depth: number;
}

export function UnifiedTreeNode({ node, depth }: Props) {
  const { expandedIds, selectedId, loadingIds, toggleExpand, selectNode } = useTreeStore();
  const isExpanded = expandedIds.has(node.id);
  const isSelected = selectedId === node.id;
  const isLoading = loadingIds.has(node.id);

  const IconComponent = Icons[node.icon.web as keyof typeof Icons] || Icons.File;

  return (
    <div>
      <div
        className={cn(
          'flex items-center gap-1 px-2 py-1 cursor-pointer hover:bg-muted',
          isSelected && 'bg-accent'
        )}
        style={{ paddingLeft: `${depth * 16}px` }}
        onClick={() => {
          selectNode(node.id);
          if (node.children) toggleExpand(node.id);
        }}
      >
        {node.children && (
          isLoading ? <Loader2 className="h-4 w-4 animate-spin" /> :
          isExpanded ? <ChevronDown className="h-4 w-4" /> :
          <ChevronRight className="h-4 w-4" />
        )}
        <IconComponent className="h-4 w-4" />
        <span>{node.label}</span>
        {node.badges?.map((badge, i) => (
          <span key={i} className="text-xs px-1 rounded" style={{ backgroundColor: badge.color }}>
            {badge.text}
          </span>
        ))}
      </div>
      {isExpanded && Array.isArray(node.children) && (
        <div>
          {node.children.map((child) => (
            <UnifiedTreeNode key={child.id} node={child} depth={depth + 1} />
          ))}
        </div>
      )}
    </div>
  );
}
```

## Troubleshooting

### Generator fails

**Symptoms**: `cargo run -- schema generate` errors out

**Solutions**:
1. Ensure Phase 2 types exist first
2. Run `cargo clean && cargo build`
3. Check YAML syntax: `cargo run -- schema validate`

### Neo4j queries fail

**Symptoms**: "Relationship type not found" or empty results

**Solutions**:
1. Verify Phase 0 migration ran:
   ```cypher
   MATCH (r:Realm)-[:HAS_LAYER]->(l) RETURN count(l)
   ```
2. Re-run migration: `pnpm infra:seed`
3. Check Neo4j logs: `docker logs novanet-neo4j`

### Icons not showing

**Symptoms**: Blank icons or fallback text

**Solutions**:
1. Check `visual-encoding.yaml` syntax
2. Verify fallback icons work:
   ```rust
   let icon = theme.icon_for("unknown").unwrap_or('?');
   ```
3. Regenerate: `cargo run -- schema generate`

### TUI crashes on startup

**Symptoms**: Panic or immediate exit

**Solutions**:
1. Check terminal supports features: `echo $TERM`
2. Run with debug: `RUST_BACKTRACE=1 cargo run -- tui`
3. Verify Neo4j is running: `cargo run -- doctor`

### Studio tree not loading

**Symptoms**: Infinite loading spinner

**Solutions**:
1. Check API endpoint: `curl http://localhost:3000/api/tree`
2. Verify CORS settings
3. Check browser console for errors
4. Ensure treeStore is initialized
