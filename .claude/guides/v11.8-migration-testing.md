# NovaNet v11.8 Migration Testing Guide

**ADR-023 & ADR-024: Class/Instance & Data Origin Traits**

Complete testing checklist and patterns for validating the v11.8 nomenclature migration.

---

## What Changed in v11.8

### ADR-023: Kind → Class, Meta → Schema

| Before | After | Category |
|--------|-------|----------|
| `NodeKind` | `NodeClass` | Type name |
| `ArcKind` | `ArcClass` | Type name |
| `KindInfo` | `ClassInfo` | Struct name |
| `:Meta:Kind` | `:Schema:Class` | Neo4j label |
| `:Meta:ArcKind` | `:Schema:ArcClass` | Neo4j label |
| `KIND_META` | `CLASS_TAXONOMY` | Constant |
| "Meta Node" | "Class" | Documentation |
| "Meta mode" | "Schema view" | UI label |

### ADR-024: Data Origin Traits

| Old Trait | New Trait | Meaning | Layer | Examples |
|-----------|-----------|---------|-------|----------|
| `invariant` | **`defined`** | Human-created once | config, structure, semantic | Page, Block, PageStructure |
| `localized` | **`authored`** | Human-written per locale | semantic, foundation | EntityContent, ProjectContent |
| `knowledge` (trait) | **`imported`** | External data brought in | knowledge | Term, Expression, Pattern |
| `derived` | **`generated`** | Produced by OUR LLM | output | PageGenerated, BlockGenerated |
| `aggregated` | **`retrieved`** | Fetched from external APIs | knowledge | GEOAnswer, SEOKeywordMetrics |

### ADR-025: Instruction Layer Renames (Bonus)

| Before | After | Function |
|--------|-------|----------|
| `PageType` | `PageStructure` | JSON defining block order |
| `PagePrompt` | `PageInstruction` | Markdown with @ references |
| `BlockPrompt` | `BlockInstruction` | Markdown with @ references |
| `[:OF_TYPE]` (Page→PageType) | `[:HAS_STRUCTURE]` | Page structure relationship |
| `[:HAS_PROMPT]` (Page→PagePrompt) | `[:HAS_INSTRUCTION]` | Page instruction relationship |

### Phase 2: Execute Migration (Green)

Update YAML, code, and tests.

#### Step 1: Update YAML Files (60 node-classes)

```bash
# Script: update-traits.sh
for file in packages/core/models/node-classes/**/*.yaml; do
    case "$(grep -m1 'trait:' "$file")" in
        *invariant*)
            sed -i 's/trait: invariant/trait: defined/' "$file"
            ;;
        *localized*)
            sed -i 's/trait: localized/trait: authored/' "$file"
            ;;
        *derived*)
            sed -i 's/trait: derived/trait: generated/' "$file"
            ;;
        *aggregated*)
            sed -i 's/trait: aggregated/trait: retrieved/' "$file"
            ;;
    esac
done

# Verify
cargo run -- schema validate
```

#### Step 3: Update Trait Parsing

```rust
// tools/novanet/src/parsers/yaml_node.rs

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "lowercase")]
pub enum NodeTrait {
    Defined,      // ← renamed from Invariant
    Authored,     // ← renamed from Localized
    Imported,     // ← renamed from Knowledge (trait)
    Generated,    // (unchanged)
    Retrieved,    // ← renamed from Aggregated
}

impl std::str::FromStr for NodeTrait {
    type Err = String;

    fn from_str(s: &str) -> Result<Self, Self::Err> {
        match s {
            "defined" => Ok(NodeTrait::Defined),
            "authored" => Ok(NodeTrait::Authored),
            "imported" => Ok(NodeTrait::Imported),
            "generated" => Ok(NodeTrait::Generated),
            "retrieved" => Ok(NodeTrait::Retrieved),
            _ => Err(format!("Invalid trait: {}", s)),
        }
    }
}
```

#[test]
fn test_trait_distribution_by_realm_layer() {
    let nodes = all_node_kinds();

#### Test 3.2: Semantic Correctness

```rust
#[test]
fn test_trait_layer_alignment() {
    // defined should be in config, structure, semantic, instruction layers
    for node in all_node_kinds() {
        if node.def.node_trait == NodeTrait::Defined {
            let valid_layers = ["config", "structure", "semantic", "instruction"];
            assert!(valid_layers.contains(&node.layer.as_str()),
                "{} has 'defined' trait but wrong layer '{}'",
                node.def.name, node.layer);
        }
    }

    // authored should be in semantic, foundation layers
    for node in all_node_kinds() {
        if node.def.node_trait == NodeTrait::Authored {
            let valid_layers = ["semantic", "foundation"];
            assert!(valid_layers.contains(&node.layer.as_str()),
                "{} has 'authored' trait but wrong layer '{}'",
                node.def.name, node.layer);
        }
    }

    // imported should be in knowledge layer
    for node in all_node_kinds() {
        if node.def.node_trait == NodeTrait::Imported {
            assert_eq!(node.layer, "knowledge",
                "{} has 'imported' trait but layer '{}'",
                node.def.name, node.layer);
        }
    }

    // generated should be in output layer
    for node in all_node_kinds() {
        if node.def.node_trait == NodeTrait::Generated {
            assert_eq!(node.layer, "output",
                "{} has 'generated' trait but layer '{}'",
                node.def.name, node.layer);
        }
    }
}
```

#### Test 3.3: Naming Pattern Consistency

```rust
#[test]
fn test_naming_patterns_match_traits() {
    for node in all_node_kinds() {
        // *Content nodes must be authored
        if node.def.name.ends_with("Content") {
            assert_eq!(node.def.node_trait, NodeTrait::Authored,
                "{} ends with 'Content' but has trait {:?}",
                node.def.name, node.def.node_trait);
        }

        // *Generated nodes must be generated in output layer
        if node.def.name.ends_with("Generated") {
            assert_eq!(node.def.node_trait, NodeTrait::Generated,
                "{} ends with 'Generated' but has trait {:?}",
                node.def.name, node.def.node_trait);
            assert_eq!(node.layer, "output");
        }

        // *Structure nodes should be defined in instruction layer
        if node.def.name.ends_with("Structure") {
            assert_eq!(node.def.node_trait, NodeTrait::Defined);
            assert_eq!(node.layer, "instruction");
        }

        // *Instruction nodes should be defined in instruction layer
        if node.def.name.ends_with("Instruction") {
            assert_eq!(node.def.node_trait, NodeTrait::Defined);
            assert_eq!(node.layer, "instruction");
        }
    }
}
```
